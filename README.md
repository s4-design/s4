## Философия

S4 построена иначе, чем большинство UI-фреймворков (дизайн систем и т.п.), которые строятся вокруг компонентов: Button, Card, Modal. Значения в них независимы: `font-size = 16px`, `padding = 12px`, `radius = 8px`.

В S4, центральный объект — **масштаб**, а не компонент. Все метрики связаны между собой через единую размерную шкалу:

```css
--size--4: 1em;                      /* базовая единица */

--font-size--md:     var(--size--4); /* = 1em */
--padding--md:       var(--size--4); /* = 1em */
--gap--md:           var(--size--4); /* = 1em */
--border-radius--md: var(--size--4); /* = 1em */
```

Система описывает не значения, а **зависимости** между ними. Изменение `font-size` на `html` изменяет не один компонент, а весь интерфейс целиком — потому что все размеры привязаны к нему через `em`.

Архитектурные инварианты S4:

1. **Три слоя `@layer`** — elements, themes, utilities — с фиксированным порядком каскада.
2. **Детерминированные формулы классов** — любой класс парсится в CSS-правило без контекста.
3. **Размерная шкала `--size--*`** — 33 именованных ступени.
4. **Device-специфичная загрузка** — каждый `{device × orientation}` имеет свой CSS-файл.
5. **Темы** — `@scope ([theme=...])` с раздельной сборкой для каждого устройства.

## AI-friendly

Архитектура S4 делает её необычно удобной для генеративных моделей (LLM).

### 1. Детерминированные формулы

Любой класс раскладывается по жёсткому паттерну:

```
d_l_display--flex        → display: flex
m_p_position--sticky     → position: sticky
color                    → var(--d_l_color, var(--color))
```

AI восстанавливает CSS из класса без контекста. Обратная задача — собрать класс из CSS — тоже однозначна.

### 2. Семантический HTML

S4 стилизует стандартные HTML-теги: `button`, `details`, `table`, `menu`, `input`, `select`, `a`, `h1`–`h6`.  `<button>Сохранить</button>` для LLM понятнее, чем `<PrimaryActionButton />`.

### 3. Параметрическое масштабирование

AI не нужно управлять каждым размером отдельно. Все метрики привязаны к `font-size` через единую шкалу `--size--*`. Изменение одного параметра пересчитывает весь интерфейс.

### 4. Минимум энтропии

Нет каталога компонентов с десятками пропсов. Есть комбинация `{элемент} + {модификатор}`. Пространство решений для AI — комбинаторика, а не перебор специализированных компонентов.

### 5. Предсказуемый каскад

`elements → themes → utilities`. AI знает: утилита всегда перебивает тему, тема — элемент. Никаких `!important`, и иных спекуляций.

<hr>

Система 4 (S4) — AI-friendly интерфейсная система. Это не AI-powered маркетинг, а следствие архитектуры: детерминированные формулы + семантический HTML + плоская модель модификаторов.

## Быстрый старт

### Установка

Скачайте [текущий релиз](https://github.com/s4-design/s4/releases/download/v.0.1.0/v0.1.0.zip):

```
s4/
├── css/
│   ├── desktop/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── themes.css
│   ├── mobile/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── themes.css
│   ├── tablet/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── themes.css
│   └── elements.css
└── js/
    ├── current-device.min.js
    └── s4.min.js
```

### Подключение

```html
<html>
    <head>
        <script src="/s4/js/s4.min.js"></script>
        <script>S4()</script>
    </head>
    <body>
        <button>Кнопка</button>
        <input type="text" placeholder="Поле ввода" />
    </body>
</html>
```

`S4()` автоматически:
- определяет устройство (desktop/tablet/mobile) и ориентацию
- загружает соответствующие CSS-файлы
- определяет тему через `prefers-color-scheme`
- устанавливает атрибут `[theme=light]` или `[theme=dark]` на `<html>`

Если нужна кастомная тема — установите `[theme="..."]` на `<html>` до вызова `S4()`.

### Структура стилей

CSS разделён на три слоя в порядке увеличения специфичности:

```
@layer elements, themes, utilities;
```

1. **elements** — стилизация HTML-элементов и кастомных тегов через `var(--*)`
2. **themes** — значения CSS-переменных под каждую тему (`@scope ([theme=light])`, `@scope ([theme=dark])`)
3. **utilities** — утилитарные классы (жёсткие значения + переменные)

## Архитектура

### Размерная система

Все метрики S4 построены на шкале `--size--*` — 33 переменных с шагом ¼em:

```
Целые четверти:  --size--0: 0em … --size--8: 2em
Половины:        --size--0_5: 0.125em … --size--7_5: 1.875em
Четверти:        --size--0_25: 0.0625em … --size--7_75: 1.9375em
```

От этой шкалы через `var()` выведены все группы:

| Группа | Пример |
|---|---|
| `--font-size--*` | `--font-size--md: var(--size--4)` |
| `--padding--*` | `--padding--md: var(--size--4)` |
| `--margin--*` | `--margin--md: var(--size--4)` |
| `--gap--*` | `--gap--md: var(--size--4)` |
| `--border-radius--*` | `--border-radius--md: var(--size--4)` |
| `--box-shadow--*` | `--box-shadow--s1: var(--size--1)` |

### Устройство и ориентация

Каждый адаптивный класс и CSS-переменная привязаны к комбинации устройства и ориентации через префикс:

| Устройство | Ориентация |
|---|---|
| `d_` — десктоп | `l_` — альбомная |
| `t_` — планшет | `p_` — портретная |
| `m_` — мобильное | |

Примеры префиксов: `d_l_` (десктоп-альбомная), `m_p_` (мобильное-портретная), `t_p_` (планшет-портретная).

JS-фреймворк (Svelte, React, Vue) подключается отдельно — S4 отвечает только за CSS и определение среды.

Кроме того, themes.css собирается отдельно для каждого устройства, поэтому --d_l_color
и --m_p_color могут иметь разные значения в рамках одной темы.

### Цвета

Акцентные цвета: `positive`, `negative`, `prime`, `second`, `success`, `danger`.

Каждый акцентный цвет имеет четыре оттенка:

| Переменная | Описание |
|---|---|
| `--{x}` | Основной цвет |
| `--{x}--light` | Светлый |
| `--{x}--dark` | Тёмный |
| `--{x}--mute` | Полупрозрачный |

Базовые: `--white`, `--black`, `--white--01`…`--white--09`, `--black--01`…`--black--09` (шаг 10% прозрачности).

## Утилитарные классы — 3 формулы

### Формула 1 — адаптивная

```
.{prefix}_{property}--{modifier}
```

Стиль привязан к устройству и ориентации.

| Класс | CSS |
|---|---|
| `d_l_display--flex` | `display: flex` |
| `m_p_position--sticky` | `position: sticky` |
| `d_l_align-content--space-around` | `align-content: space-around` |

С псевдоклассом: `d_l_background-color--transparent\:hover:hover`

С псевдоэлементом: `d_l_display--none\:\:before::before`

### Формула 2 — единая

```
.{property}--{modifier}
```

Стиль един для всех устройств и ориентаций.

| Класс | CSS |
|---|---|
| `border--collapse` | `border-collapse: collapse` |
| `box-sizing--border-box` | `box-sizing: border-box` |

### Формула 3 — переменная

```
.{property}
```

Подключает CSS-свойство к цепочке переменных темы:

```css
color: var(--d_l_color, var(--color));
```

| Класс | Результат |
|---|---|
| `.color` | `color: var(--d_l_color, var(--color))` |
| `.background-color` | `background-color: var(--d_l_background-color, var(--background-color))` |

С псевдоклассом: `.color\:hover:hover`

Без Formula 3 значение свойства было бы жёстко задано в CSS. Formula 3 делает его контекстно-зависимым — тема на устройстве решает, каким оно будет. Один и тот же элемент может иметь разный `color` на десктопе и мобильном, в светлой и тёмной теме, без изменения HTML.

## Элементы

HTML-элементы и кастомные теги S4 стилизуются через CSS-переменные — ни одного жёсткого значения в `elements.css`. Каждый кастомный элемент `<e-{name}>` имеет класс-дубликат `.element--{name}` для сред без кастомных элементов (React, legacy).

| Кастомный тег | Класс-дубликат | Описание |
|---|---|---|
| `<e-badge>` | `.element--badge` | Бейдж |
| `<e-icon>` | `.element--icon` | Иконка |
| `<e-popover>` | `.element--popover` | Поповер |
| `<e-message>` | `.element--message` | Сообщение |
| `<e-truncate>` | `.element--truncate` | Усечение текста |
| `<e-group>` | `.element--group` | Группировка |
| `<e-line>` | `.element--line` | Разделитель |
| `<button>` | `.element--button` | Кнопка |
| `<input>` | `.element--input` | Поле ввода |
| `<select>` | `.element--select` | Выбор |
| `<details>` | `.element--details` | Детали |
| `<menu>` | `.element--menu` | Меню |

**Правила:**

1. **Один селектор на ноду.** Используй либо `<e-{name}>`, либо `<tag class="element--{name}">`. Никогда оба одновременно.
2. **Специфичность.** Класс `.element--name` (0,1,0,0) — наравне с утилитами S4. Кастомный тег `<e-name>` (0,0,1,0) — базовый слой. Любой утилитарный класс переопределяет стили элемента без `!important`.

Базовый HTML уже стилизован — классы не обязательны:

```html
<button>Нажать</button>
<input type="text" placeholder="Введите текст" />
<table>
    <tr><th>Имя</th><th>Значение</th></tr>
    <tr><td>Параметр</td><td>Результат</td></tr>
</table>
```

## Темы

S4 определяет тему через `prefers-color-scheme` и устанавливает атрибут `[theme]` на `<html>`:

```css
@scope ([theme=light]) { /* светлые значения */ }
@scope ([theme=dark])  { /* тёмные значения */ }
```

**Device-специфичность тем:** каждая тема собирается отдельно для каждого устройства — `desktop/themes.css`, `mobile/themes.css`, `tablet/themes.css`. Значения `[theme=dark]` на десктопе могут отличаться от `[theme=dark]` на мобильном. Равенство между устройствами не гарантировано.

Для device-независимого стиля используйте Formula 2 или явные значения.

Переключение темы в рантайме отслеживается автоматически.
