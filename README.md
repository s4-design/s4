## Философия

С4 - **среда для построения дизайн-систем**, а не фреймворк с каталогом компонентов. Ядро С4 даёт инструмент: трёхслойный каскад `@layer`, детерминированные формулы классов, device-специфичную загрузку. Пресеты задают конфигурацию интерфейса - визуальную идентичность и поведение. Дизайн-система формируется из ядра С4, пресета и пользовательских компонентов и паттернов.

Default-пресеты построены иначе, чем большинство UI-фреймворков, которые строятся вокруг компонентов: Button, Card, Modal. Значения в них независимы: font-size = 16px, padding = 12px, radius = 8px.
Пресеты light/dark - готовые конфигурации с EM-центричной параметрической моделью, где все метрики выведены из единой шкалы.

Каждый пресет - самостоятельная конфигурация: light и dark могут совпадать в метриках сегодня, но ничто не мешает им различаться полностью завтра.

Центральный объект default-пресетов - **гравитация** (относительный масштаб), а не компонент. Все метрики связаны между собой через размерную шкалу:

```css
--size--4: 1em;                      /* базовая единица */

--font-size--md:     var(--size--4); /* = 1em */
--padding--md:       var(--size--4); /* = 1em */
--gap--md:           var(--size--4); /* = 1em */
--border-radius--md: var(--size--4); /* = 1em */
```

Система описывает не значения, а **зависимости** между ними. Изменение `font-size` на `html` изменяет не один компонент, а весь интерфейс целиком - потому что все размеры привязаны к нему через `em`.

Архитектурные инварианты С4:

1. **Три слоя `@layer`** - elements, presets, utilities - с фиксированным порядком каскада.
2. **Детерминированные формулы классов** - любой класс парсится в CSS-правило без контекста.
3. **Размерная шкала `--size--*`** - 3 группы именованных ступеней (переменных) с шагом ¼.
4. **Device-специфичная загрузка** - каждый `{device × orientation}` имеет свой CSS-файл.
5. **Пресеты** - конфигурации интерфейса, изолированные через `@scope ([preset=...])`, с раздельной сборкой для каждого устройства.

## AI-friendly

Архитектура С4 делает её необычно удобной для генеративных моделей (LLM).

### 1. Детерминированные формулы

Любой класс раскладывается по жёсткому паттерну:

```
d_l_display--flex        → display: flex
m_p_position--sticky     → position: sticky
color                    → var(--d_l_color, var(--color))
```

AI восстанавливает CSS из класса без контекста. Обратная задача - собрать класс из CSS - тоже однозначна.

### 2. Семантический HTML

С4 стилизует стандартные HTML-теги: `button`, `details`, `table`, `menu`, `input`, `select`, `a`, `h1`–`h6`. `<button>Сохранить</button>` для LLM понятнее, чем `<PrimaryActionButton />`.

### 3. Параметрическое масштабирование

AI не нужно управлять каждым размером отдельно. Все метрики привязаны к `font-size` через единую шкалу `--size--*`. Изменение одного параметра пересчитывает весь интерфейс.

### 4. Минимум энтропии

Нет каталога компонентов с десятками пропсов. Есть комбинация `{элемент} + {модификатор}`. Пространство решений для AI - комбинаторика, а не перебор специализированных компонентов.

### 5. Предсказуемый каскад

`elements → presets → utilities`. Утилита всегда перебивает пресет, пресет - элемент. Никаких `!important`, никаких спекуляций.

<hr>

Система 4 (С4) - AI-friendly интерфейсная система. Это не маркетинговый ярлык, а следствие архитектуры: детерминированные формулы + семантический HTML + плоская модель модификаторов.

## Быстрый старт

### Установка

Скачайте [текущий релиз](https://github.com/s4-design/s4/releases/):

```
s4/
├── css/
│   ├── desktop/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── config.css
│   ├── mobile/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── config.css
│   ├── tablet/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── config.css
│   └── elements.css
└── js/
    ├── device-state.min.js
    └── s4.min.js
```

### Подключение

```html
<html>
    <head>
        <script src="/s4/js/s4.min.js"></script>
        <script>S4()</script>
    </head>
    <body>
        <button>Кнопка</button>
        <input type="text" placeholder="Поле ввода" />
    </body>
</html>
```

`S4()` автоматически:
- определяет устройство (desktop/tablet/mobile) и ориентацию
- загружает соответствующие CSS-файлы
- определяет пресет через `prefers-color-scheme`
- устанавливает атрибут `[preset=light]` или `[preset=dark]` на `<html>`

Если нужен кастомный пресет - установите `[preset="..."]` на `<html>` до вызова `S4()`.

### Структура стилей

CSS разделён на три слоя в порядке увеличения специфичности:

```
@layer elements, presets, utilities;
```

1. **elements** - стилизация HTML-элементов и кастомных тегов через `var(--*)`
2. **presets** - значения CSS-переменных, заданные в пресете (`@scope ([preset=light])`, `@scope ([preset=dark])`)
3. **utilities** - утилитарные классы (жёсткие значения + переменные)

## Архитектура

### Размерная шкала

Все метрики default-пресетов построены на шкале `--size--*` - 3 группы переменных с шагом ¼em:

```
Целые:    --size--0: 0em … --size--8: 2em
Половины: --size--0_5: 0.125em … --size--7_5: 1.875em
Четверти: --size--0_25: 0.0625em … --size--7_75: 1.9375em
```

От этой шкалы через `var()` выведены все группы:

| Группа | Пример |
|---|---|
| `--font-size--*` | `--font-size--md: var(--size--4)` |
| `--padding--*` | `--padding--md: var(--size--4)` |
| `--margin--*` | `--margin--md: var(--size--4)` |
| `--gap--*` | `--gap--md: var(--size--4)` |
| `--border-radius--*` | `--border-radius--md: var(--size--4)` |
| `--box-shadow--*` | `--box-shadow--s1: var(--size--1)` |

Единицы измерения (em, rem, px и т.д.) задаются в пресете. Default-пресеты используют `em`. Кастомный пресет может задать произвольные значения.

### Устройство и ориентация

Каждый адаптивный класс и CSS-переменная привязаны к комбинации устройства и ориентации через префикс:

| Устройство | Ориентация |
|---|---|
| `d_` - десктоп | `l_` - альбомная |
| `t_` - планшет | `p_` - портретная |
| `m_` - мобильное | |

Примеры префиксов: `d_l_` (десктоп, альбомная), `m_p_` (мобильное, портретная), `t_p_` (планшет, портретная).

CSS-переменные тоже могут быть адаптивными. Formula 3 строит цепочку fallback:

```css
var(--d_l_color, var(--color))
```

где первый аргумент - значение для конкретного `{device × orientation}`, второй - общее.

Кроме того, config.css собирается отдельно для каждого устройства, поэтому `--d_l_color` и `--m_p_color` могут иметь разные значения в рамках одного пресета.

JS-фреймворк (Svelte, React, Vue) подключается отдельно - С4 отвечает только за CSS и определение среды.

### Цвета

Акцентные цвета: `positive`, `negative`, `prime`, `second`, `success`, `danger`.

Каждый акцентный цвет имеет четыре оттенка:

| Переменная | Описание |
|---|---|
| `--{x}` | Основной цвет |
| `--{x}--light` | Светлый |
| `--{x}--dark` | Тёмный |
| `--{x}--mute` | Полупрозрачный |

Базовые: `--white`, `--black`, `--white--01`…`--white--09`, `--black--01`…`--black--09` (шаг 10% прозрачности).

## Утилитарные классы - 3 формулы

### Формула 1 - адаптивная

```
.{prefix}_{property}--{modifier}
```

Стиль привязан к устройству и ориентации.

| Класс | CSS |
|---|---|
| `d_l_display--flex` | `display: flex` |
| `m_p_position--sticky` | `position: sticky` |
| `d_l_align-content--space-around` | `align-content: space-around` |

С псевдоклассом: `d_l_background-color--transparent\:hover:hover`

С псевдоэлементом: `d_l_display--none\:\:before::before`

### Формула 2 - единая

```
.{property}--{modifier}
```

Стиль един для всех устройств и ориентаций.

| Класс | CSS |
|---|---|
| `border--collapse` | `border-collapse: collapse` |
| `box-sizing--border-box` | `box-sizing: border-box` |

### Формула 3 - переменная

```
.{property}
```

Подключает CSS-свойство к цепочке переменных пресета:

```css
color: var(--d_l_color, var(--color));
```

| Класс | Результат |
|---|---|
| `.color` | `color: var(--d_l_color, var(--color))` |
| `.background-color` | `background-color: var(--d_l_background-color, var(--background-color))` |

С псевдоклассом: `.color\:hover:hover`

Без Formula 3 значение свойства было бы жёстко задано в CSS. Formula 3 делает его контекстно-зависимым - пресет на устройстве решает, каким оно будет. Один и тот же элемент может иметь разный `color` на десктопе и мобильном, в светлом и тёмном пресете, без изменения HTML.

## Элементы

HTML-элементы и кастомные теги С4 стилизуются через CSS-переменные - ни одного жёсткого значения в `elements.css`. Каждый кастомный элемент `<e-{name}>` имеет класс-дубликат `.element--{name}` для сред без кастомных элементов (React, legacy).

| Кастомный тег | Класс-дубликат | Описание |
|---|---|---|
| `<e-badge>` | `.element--badge` | Бейдж |
| `<e-icon>` | `.element--icon` | Иконка |
| `<e-popover>` | `.element--popover` | Поповер |
| `<e-message>` | `.element--message` | Сообщение |
| `<e-truncate>` | `.element--truncate` | Усечение текста |
| `<e-group>` | `.element--group` | Группировка |
| `<e-line>` | `.element--line` | Разделитель |
| `<button>` | `.element--button` | Кнопка |
| `<input>` | `.element--input` | Поле ввода |
| `<select>` | `.element--select` | Выбор |
| `<details>` | `.element--details` | Детали |
| `<menu>` | `.element--menu` | Меню |

**Правила:**

1. **Один селектор на ноду.** Используй либо `<e-{name}>`, либо `<tag class="element--{name}">`. Никогда оба одновременно.
2. **Специфичность.** Класс `.element--name` (0,1,0,0) - наравне с утилитами С4. Кастомный тег `<e-name>` (0,0,1,0) - базовый слой. Любой утилитарный класс переопределяет стили элемента без `!important`.

Базовый HTML уже стилизован - классы не обязательны:

```html
<button>Нажать</button>
<input type="text" placeholder="Введите текст" />
<table>
    <tr><th>Имя</th><th>Значение</th></tr>
    <tr><td>Параметр</td><td>Результат</td></tr>
</table>
```

## Пресеты

Пресет С4 - **конфигурация интерфейса**, определяющая его визуальную идентичность и поведение. Пресет задаёт не только метрики (`--size--*`) и цвета, но и типографику (`--font-family`, `--font-weight`, `--line-height`), анимации (`--transition-duration`, `--transition-timing-function`), поведение (`--overscroll-behavior`, `--word-break`, `--hyphens`), и любые другие CSS-переменные, необходимые проекту.

Пресет - слой дизайн-системы, не вся система. Дизайн-система = С4 (ядро: каскад, формулы, device-загрузка) + пресет (конфигурация интерфейса) + компоненты и паттерны.

Любая переменная, объявленная в пресете (`--{name}`), автоматически становится доступна как модификатор в xtra-утилитах Формулы 1. Например, если пресет задаёт `--brand-gradient: linear-gradient(...)`, то класс `d_l_background--brand-gradient` будет работать.

С4 определяет пресет через `prefers-color-scheme` и устанавливает атрибут `[preset]` на `<html>`:

```css
@scope ([preset=light]) { /* светлый пресет */ }
@scope ([preset=dark])  { /* тёмный пресет */ }
```

Каждый блок `@scope` содержит все переменные пресета для данного устройства. Пример содержимого `:scope`:
```css
:scope {
    --position: relative;
    --background-color: var(--positive--light);
    --color: var(--negative);
    --tab-size: 4;
    --font-size: var(--font-size--md);
    --font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans";
    --font-weight: normal;
    --line-height: var(--size--6);
    --text-underline-offset: var(--size--0_25);
    --overscroll-behavior: none;
    --word-break: normal;
    --hyphens: auto;
    --transition-duration: .15s;
    --transition-timing-function: linear;
}
```

**Device-специфичность пресетов:** каждый пресет собирается отдельно для каждого устройства - `desktop/config.css`, `mobile/config.css`, `tablet/config.css`. Значения `[preset=dark]` на десктопе могут отличаться от `[preset=dark]` на мобильном. Равенство между устройствами не гарантировано.

**Кастомный пресет - другая конфигурация интерфейса.** Установи `[preset="my-name"]` на `<html>` до вызова `S4()` и напиши CSS:

```css
@scope ([preset=my-name]) {
    :scope {
        --size--4: 1.25em;
        --padding--md: 2em;
        --color: var(--black--07);
        --font-family: "IBM Plex Sans", system-ui;
        --transition-duration: 0.2s;
    }
}
```

Для device-независимого стиля используйте Formula 2 или явные значения. Пресеты влияют только на переменные - утилитарные классы Formula 1 и 2 не зависят от пресета.

Переключение пресета в рантайме отслеживается через `matchMedia('prefers-color-scheme')`.
