# С4 - Кодо-ориентированная среда для построения дизайн-систем

Интерфейс определяется комбинацией `{device × orientation × preset}`, а не каталогом компонентов. Ядро: трёхслойный каскад `@layer`, детерминированные формулы классов, размерная шкала `--size--*`. Пресеты - конфигурации интерфейса: метрики, цвета, типографика, анимации, поведение. Дизайн-система = С4 + пресет + компоненты и паттерны.

Этот файл - техническое описание для разработчиков и AI-агентов. Он описывает систему как она есть, включая архитектурные инварианты, формулы, соглашения и механику.

---

## Философия

С4 - **среда для построения дизайн-систем**, а не фреймворк с каталогом компонентов. Ядро даёт инструмент: каскад `@layer`, формулы классов, device-специфичную загрузку. Пресеты задают конфигурацию интерфейса - визуальную идентичность и поведение. Дизайн-система формируется из ядра С4, пресета и пользовательских компонентов и паттернов.

**Архитектурные инварианты С4:**

1. **Три слоя `@layer`** - elements, presets, utilities - с фиксированным порядком каскада.
2. **Детерминированные формулы классов** - любой класс парсится в CSS-правило без контекста.
3. **Размерная шкала `--size--*`** - 3 группы именованных ступеней с шагом ¼.
4. **Device-специфичная загрузка** - каждый `{device × orientation}` имеет свой CSS-файл.
5. **Пресеты** - конфигурации интерфейса (метрики, цвета, типографика, анимации, поведение), собираемые отдельно для каждого устройства.

Default-пресеты (light/dark) - готовые конфигурации с EM-центричной моделью. Каждый пресет - самостоятельная конфигурация: light и dark могут совпадать в метриках сегодня, но ничто не мешает им различаться полностью. Кастомный пресет - другая конфигурация, на основе которой формируется новая дизайн-система.

## Архитектура

### Три слоя CSS

```
@layer elements, presets, utilities;
```

Каждый следующий слой доминирует над предыдущим:

1. **elements** - стилизация HTML-элементов и кастомных тегов через `var(--*)`. Ни одного жёсткого значения. Загружается один раз (`elements.css`).
2. **presets** - значения CSS-переменных, заданные в пресете (`@scope ([preset=light])`, `@scope ([preset=dark])`). Загружается для каждого устройства отдельно (`desktop/config.css`, `mobile/config.css`).
3. **utilities** - утилитарные классы (жёсткие значения + Formula 3 с переменными).

### Размерная шкала

Default-пресеты определяют 33 переменные `--size--*` с бисекционным шагом ¼em:

```
Целые:    --size--0: 0em   --size--1: 0.25em   …   --size--8: 2em
Половины: --size--0_5: 0.125em … --size--7_5: 1.875em
Четверти: --size--0_25: 0.0625em … --size--7_75: 1.9375em
```

`--size--4: 1em` - опорная точка шкалы. От неё через `var()` выведены все метрические группы.

Единицы измерения (em, rem, px и т.д.) задаются в пресете. Default-пресеты используют `em`.

От шкалы через `var()` выведены группы - это конвенция default-пресетов, а не архитектурное ограничение:

| Группа | Пример (default-пресет) |
|---|---|
| `--font-size--*` | `--font-size--md: var(--size--4)` |
| `--padding--*` | `--padding--md: var(--size--4)` |
| `--margin--*` | `--margin--md: var(--size--4)` |
| `--gap--*` | `--gap--md: var(--size--4)` |
| `--border-radius--*` | `--border-radius--md: var(--size--4)` |
| `--box-shadow--*` | `--box-shadow--s1: var(--size--1)` |

В default-пресетах работает правило: `--{group}--{mod} = var(--size--{N})`. Кастомный пресет может задать произвольные значения.

### Устройство и ориентация

Префикс класса или переменной = устройство + ориентация:

| Устройство | Ориентация |
|---|---|
| `d_` - десктоп | `l_` - альбомная |
| `t_` - планшет | `p_` - портретная |
| `m_` - мобильное | |

Префикс всегда двухсимвольный: `d_l_`, `m_p_`, `t_l_` и т.д.

CSS-переменные тоже могут быть адаптивными. Formula 3 строит цепочку fallback:

```css
var(--d_l_color, var(--color))
```

где первый аргумент - значение для конкретного `{device × orientation}`, второй - общее.

### Цвета

Акцентные цвета: `positive`, `negative`, `prime`, `second`, `success`, `danger`.

Каждый имеет четыре оттенка:

| Переменная | Описание |
|---|---|
| `--{x}` | Основной |
| `--{x}--light` | Светлый |
| `--{x}--dark` | Тёмный |
| `--{x}--mute` | Полупрозрачный (45%) |

Базовые: `--white`, `--black`, `--white--01`…`--white--09`, `--black--01`…`--black--09` (шаг 10% прозрачности).

## Подключение (быстрый старт)

### HTML

```html
<html>
    <head>
        <script src="/s4/js/s4.min.js"></script>
        <script>С4()</script>
    </head>
    <body>
        <button>Кнопка</button>
    </body>
</html>
```

### Структура файлов

```
s4/
├── css/
│   ├── desktop/
│   │   ├── landscape.css    - утилиты для d_l_
│   │   ├── portrait.css     - утилиты для d_p_
│   │   └── config.css       - значения --* для [preset=light/dark] на десктопе
│   ├── mobile/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── config.css       - значения --* для мобильного
│   ├── tablet/
│   │   ├── landscape.css
│   │   ├── portrait.css
│   │   └── config.css       - значения --* для планшета
│   └── elements.css         - слой elements (один на все устройства)
└── js/
    ├── current-device.min.js
    └── s4.min.js
```

### Что делает С4()

1. Устанавливает атрибут `[preset=light]` или `[preset=dark]` на `<html>` на основе `prefers-color-scheme` (если пресет не задан вручную).
2. Определяет устройство и ориентацию через `current-device`.
3. Грузит `elements.css` (один раз).
4. Грузит `{device}/config.css`.
5. Грузит `{device}/{orientation}.css` согласно `dependency-map.json`.
6. Следит за изменением ориентации и пресета в рантайме.

## Утилитарные классы - 3 формулы

### Формула 1 - адаптивная

```
.{prefix}_{property}--{modifier}
```
#### Расширенные

С псевдоклассом:
```
.{prefix}_{property}--{modifier}:{pseudoClass}
```

С псевдоэлементом:
```
.{prefix}_{property}--{modifier}::{pseudoElement}
```

С псевдоклассом + псевдоэлементом:
```
.{prefix}_{property}--{modifier}:{pseudoClass}::{pseudoElement}
```

Стиль привязан к устройству и ориентации.

| Класс | CSS |
|---|---|
| `d_l_display--flex` | `display: flex` |
| `m_p_position--sticky` | `position: sticky` |
| `d_l_align-content--space-around` | `align-content: space-around` |

С псевдоклассом: `d_l_background-color--transparent\:hover:hover`

С псевдоэлементом: `d_l_display--none\:\:before::before`

В xtra-слое (presets) модификатор - это имя CSS-переменной пресета, а не жёсткое значение:
```css
d_l_color--black { color: var(--black); }
d_l_font-size--md { font-size: var(--font-size--md); }
```
Любая переменная, объявленная в пресете, автоматически становится доступным модификатором. Например, если пресет задаёт `--brand-gradient: linear-gradient(...)`, то класс `d_l_background--brand-gradient` будет работать.

### Формула 2 - единая

```
.{property}--{modifier}
```

Стиль един для всех устройств и ориентаций.

| Класс | CSS |
|---|---|
| `border--collapse` | `border-collapse: collapse` |
| `box-sizing--border-box` | `box-sizing: border-box` |

Псевдокласс: `.border--collapse\:hover:hover`

Псевдоэлемент: `.border--collapse\:\:before::before`

### Формула 3 - переменная

```
.{property}
```

Подключает CSS-свойство к цепочке переменных пресета:

```css
property: var(--{prefix}_{property}, var(--{property}));
```

| Класс | Результат |
|---|---|
| `.color` | `color: var(--d_l_color, var(--color))` |
| `.background-color` | `background-color: var(--d_l_background-color, var(--background-color))` |

Псевдокласс: `.color\:hover:hover` → `var(--d_l_color\:hover, var(--color\:hover))`

Псевдоэлемент: `.color\:\:before::before` → `var(--d_l_color\:\:before, var(--color\:\:before))`

Formula 3 делает свойство контекстно-зависимым. Один и тот же элемент может иметь разный `color` на десктопе и мобильном, в светлом и тёмном пресете, без изменения HTML. Значение определяется не CSS-правилом, а цепочкой переменных: сначала проверяется device-специфичное значение, затем общее.

## Элементы

HTML-элементы и кастомные теги С4 стилизуются через CSS-переменные. В `elements.css` нет жёстких значений - только `var(--*)`. Значения приходят из `config.css`.

Каждый кастомный элемент `<e-{name}>` имеет класс-дубликат `.element--{name}` для сред, где кастомные элементы недоступны (React, legacy). CSS-правило пишется через оба селектора:

```css
<e-{name}>,
.element--{name} {
    ...
}
```

| Тег | Класс-дубликат | Описание |
|---|---|---|
| `<e-badge>` | `.element--badge` | Бейдж |
| `<e-icon>` | `.element--icon` | Иконка |
| `<e-popover>` | `.element--popover` | Поповер с телом |
| `<e-message>` | `.element--message` | Сообщение с header/body/footer |
| `<e-truncate>` | `.element--truncate` | Усечение текста |
| `<e-group>` | `.element--group` | Группировка элементов |
| `<e-line>` | `.element--line` | Разделитель |
| `<button>` | `.element--button` | Кнопка (стилизованный HTML) |
| `<input>` | `.element--input` | Поле ввода |
| `<select>` | `.element--select` | Выбор |
| `<details>` | `.element--details` | Раскрывающийся блок |
| `<menu>` | `.element--menu` | Меню |

**Правила:**

1. **Один селектор на ноду.** Используй либо `<e-{name}>`, либо `<tag class="element--{name}">`. Никогда оба одновременно.
2. **Специфичность.** Класс `.element--name` (0,1,0,0) - наравне с утилитами С4. Кастомный тег `<e-name>` (0,0,1,0) - базовый слой. Любой утилитарный класс переопределяет стили элемента без `!important`.

Базовый HTML уже стилизован - классы не обязательны:

```html
<button>Нажать</button>
<input type="text" placeholder="Введите текст" />
<table>
    <tr><th>Имя</th><th>Значение</th></tr>
    <tr><td>Параметр</td><td>Результат</td></tr>
</table>
```

## Пресеты

Пресет С4 - **конфигурация интерфейса**, определяющая его визуальную идентичность и поведение. Пресет задаёт не только метрики (`--size--*`) и цвета, но и типографику (`--font-family`, `--font-weight`, `--line-height`, `--text-underline-offset`), анимации (`--transition-duration`, `--transition-timing-function`), поведение (`--overscroll-behavior`, `--word-break`, `--hyphens`, `--tab-size`), и любые другие CSS-переменные, необходимые проекту.

Пресет - слой дизайн-системы, не вся система. Дизайн-система = С4 (ядро: каскад, формулы, device-загрузка) + пресет (конфигурация интерфейса) + компоненты и паттерны.

Любая переменная, объявленная в пресете (`--{name}`), становится доступна как модификатор в xtra-утилитах Формулы 1. Например, если пресет задаёт `--brand-gradient: linear-gradient(...)`, то класс `d_l_background--brand-gradient` будет работать.

С4 определяет пресет через `prefers-color-scheme` и устанавливает атрибут `[preset]` на `<html>`. Если атрибут уже установлен до вызова `С4()` - значение сохраняется.

Пресеты изолированы через `@scope`:

```css
@scope ([preset=light]) { /* светлый пресет */ }
@scope ([preset=dark])  { /* тёмный пресет */ }
```

Каждый `@scope`-блок содержит все переменные пресета для данного устройства. Пример содержимого `:scope`:
```css
:scope {
    --position: relative;
    --background-color: var(--positive--light);
    --color: var(--negative);
    --tab-size: 4;
    --font-size: var(--font-size--md);
    --font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans";
    --font-weight: normal;
    --line-height: var(--size--6);
    --text-underline-offset: var(--size--0_25);
    --overscroll-behavior: none;
    --word-break: normal;
    --hyphens: auto;
    --transition-duration: .15s;
    --transition-timing-function: linear;
}
```

**Изоляция пресетов.** Пресет применяется к элементу с соответствующим атрибутом `[preset=...]`, независимо от вложенности. `<div preset="light">` может содержать `<div preset="dark">` — родительский пресет не влияет на дочерний и наоборот. Пресеты можно менять независимо от DOM-иерархии.

**Device-специфичность.** Каждый пресет собирается отдельно для каждого устройства (`desktop/config.css`, `mobile/config.css`, `tablet/config.css`). Значения `[preset=dark]` на десктопе могут отличаться от `[preset=dark]` на мобильном. Равенство между устройствами не гарантировано.

**Кастомный пресет - другая конфигурация интерфейса.** Установи `[preset="my-name"]` на `<html>` до вызова `С4()` и напиши CSS:

```css
@scope ([preset=my-name]) {
    :scope {
        --size--4: 1.25rem;
        --padding--md: 2rem;
        --color: oklch(0.2 0.02 260);
        --font-family: "IBM Plex Sans", system-ui;
        --transition-duration: 0.2s;
    }
}
```

**Иконографика, шрифты, прочие ресурсы** - вне ответственности С4. Пользователь дизайн-системы подключает их отдельно, так, как нужно его сборке.

**Для device-независимого стиля** используйте Formula 2 или явные значения в CSS. Пресеты влияют только на переменные - утилитарные классы Formula 1 и 2 не зависят от пресета.

Переключение пресета в рантайме отслеживается через `matchMedia('prefers-color-scheme')`.

## AI-friendly

Архитектура С4 делает её необычно удобной для генеративных моделей:

### 1. Детерминированные формулы

Любой класс раскладывается по жёсткому паттерну:

```
d_l_display--flex    → display: flex
m_p_position--sticky → position: sticky
color                → var(--d_l_color, var(--color))
```

AI восстанавливает CSS из класса без контекста. Обратная задача - собрать класс из CSS - тоже однозначна.

### 2. Семантический HTML

С4 стилизует стандартные HTML-теги: `button`, `details`, `table`, `menu`, `input`, `select`, `a`, `h1`–`h6`. LLM обучены на миллиардах HTML-документов. `<button>Сохранить</button>` для них понятнее, чем `<PrimaryActionButton />`.

### 3. Параметрическое масштабирование

AI не нужно управлять каждым размером отдельно. В default-пресетах все метрики привязаны к `--size--*`. Изменение одного параметра пересчитывает весь интерфейс.

### 4. Минимум энтропии

Нет каталога компонентов с десятками пропсов. Есть комбинация `{элемент} + {модификатор}`. Пространство решений для AI - комбинаторика, а не перебор специализированных компонентов.

### 5. Предсказуемый каскад

```
elements → presets → utilities
```

Утилита всегда перебивает пресет, пресет - элемент. Никаких `!important`, никаких спекуляций. AI знает, какой слой победит, без вычисления специфичности.

## dependency-map.json

JS-рантайм грузит CSS на основе `dependency-map.json`. Дефолтная карта:

```json
{
    "desktop": {
        "landscape": [{ "desktop": "landscape" }],
        "portrait":  [{ "desktop": "portrait" }]
    },
    "tablet": {
        "landscape": [{ "tablet": "landscape" }],
        "portrait":  [{ "tablet": "portrait" }]
    },
    "mobile": {
        "landscape": [{ "mobile": "landscape" }],
        "portrait":  [{ "mobile": "portrait" }]
    }
}
```

Если файл не найден - используется встроенный fallback с той же структурой.

Пользователь может переопределить карту, чтобы добавить свои CSS-файлы в каскад или изменить приоритет загрузки для конкретного `{device × orientation}`.

Пример карты зависимостей (mobile-first):

```json
{
    "desktop": {
        "landscape": [
            {"desktop": "landscape"}
        ],
        "portrait": [
            {"desktop": "landscape"},
            {"desktop": "portrait"}
        ]
    },
    "tablet": {
        "landscape": [
            {"desktop": "landscape"},
            {"tablet": "landscape"}
        ],
        "portrait": [
            {"desktop": "landscape"},
            {"desktop": "portrait"},
            {"tablet": "landscape"},
            {"tablet": "portrait"}
        ]
    },
    "mobile": {
        "landscape": [
            {"desktop": "landscape"},
            {"mobile": "portrait"},
            {"mobile": "landscape"}
        ],
        "portrait": [
            {"desktop": "landscape"},
            {"desktop": "portrait"},
            {"mobile": "portrait"}
        ]
    }
}
```
