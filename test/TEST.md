# Тестирование S4

## Структура

```
test/
├── TEST.md               ← этот файл
├── snapshot.mjs           ← скрипт автоматических тестов
├── index.html             ← страница со всеми элементами S4
├── icons/                 ← иконки для тестовой страницы
├── __snapshots__/         ← эталонные скриншоты (в Git)
│   ├── light/             ←   для [theme=light]
│   └── dark/              ←   для [theme=dark]
└── __diff__/              ← диффы при расхождениях (в .gitignore)
    ├── light/
    └── dark/
```

## Автоматические тесты (скриншотные снэпшоты)

Playwright открывает `test/index.html` в headless-браузере, скриншотит каждый элемент по `id` и сравнивает с эталоном.

### Запуск

```bash
pnpm _dev-test              # сравнить с эталонами
pnpm _dev-test-update       # перезаписать эталоны (новые скриншоты)
```

Скрипт сам поднимает сервер (`pnpx serve .` на порту 3000), не нужно ничего запускать отдельно.

### Что проверяется

Для каждой темы (`light`, `dark`) проход по всем `id`:
- 36 элементов видимы — делается скриншот
- 5 элементов datalist скрыты — пропускаются
- Итого: 72 скриншота (36 × 2 темы)

### Результат

```
=== Results: 72 passed, 0 failed, 10 skipped (light: 36 snapshots, dark: 36 snapshots) ===
```

- `PASS` — совпало с эталоном
- `FAIL` — отличается (дифф в `__diff__/{theme}/{id}.png`)
- `SKIP` — элемент не найден или скрыт
- `SAVE` — эталон сохранён (первый запуск или `--update`)

### Когда обновлять эталоны

После изменения стилей элементов — запустить `pnpm _dev-test-update`. Закоммитить обновлённые скриншоты вместе с изменениями CSS.

### Команды package.json

```json
"_dev-test": "node test/snapshot.mjs",
"_dev-test-update": "node test/snapshot.mjs --update"
```

## Ручной просмотр тестовой страницы

```bash
pnpx serve .
# открыть http://localhost:3000/test/
```

Для визуальной проверки изменений, отладки, разработки новых вариаций элементов.
